# 活动运营产品规格

> 状态：**产品定稿（待工程排期）**  
> 更新日期：2026-07-23  
> 相关：首页 Hero B / 今日推荐、共读群、灵修系列（可选连接）

本文整合活动运营能力：**群定向曝光、强模板自配落地页、积木控件（含互动/报名）、多日日课、分享讨论**；场景覆盖读经、代祷、聚会、迎新、节期、见证、服事招募与习惯挑战（§8）。不含意见治理、宣教外展专题。  
不以绑定已上架系列为主；系列仅为可选跳转。

---

## 1. 定位

| | 说明 |
|--|------|
| **是什么** | 群主/管理员为负责的群配置限时活动：定谁能看、出现在首页何处、点进去读什么、如何互动与分享 |
| **典型场景** | 手里有 **N 天资料**（如 10 日），经首页发给指定群，引导按天阅读，并分享、讨论 |
| **不是什么** | 群内任务/IM 的替代；全站广告台；自由 HTML 建站；必须以内容库系列为前提 |

**成功标准**

1. 群主不依赖上架系列，也能发出完整活动  
2. 非受众用户首页零打扰  
3. **一份多日资料可在单个活动内配完**，并支持阅读进度、分享、讨论  

---

## 2. 原则

1. **自配是第一公民** — 活动页内容由运营填写；模板降低门槛，不替代创作  
2. **最小可见优先** — 默认指定群；全站仅平台超管  
3. **配置不进群** — 群页/群设置不增加活动入口  
4. **曝光按人过滤** — 仅命中受众才插入 Hero / 今日推荐  
5. **交互靠控件扩展** — 新玩法 = 新控件/新模板，不是每次改业务代码  
6. **长系列可选** — 平台内容库系列可被控件连接，非配置主路径  

---

## 3. 模型

```text
活动 Campaign
 ├─ 基础：名称、状态、创建者、时段
 ├─ 受众：多选群（并集）；默认非全站
 ├─ 曝光 Placements
 │    ├─ 今日推荐 · 第 1/2/3 位（群主可用）
 │    └─ Hero B（仅超管）
 └─ 落地 Landing
      ├─ 模板（场景/版式/我的）
      ├─ 积木页（内容 / 圣经 / 行动 / 多日 / 互动 / 分享）
      └─ （可选）另存为我的模板

平台内容库 SeriesPackage（可选）
 └─ 仅被「系列入口」控件引用
```

- 一个活动 = **一个落地页**；多处曝光共用  
- 多日资料 = 活动内的 **日课控件数据**，不是多个活动  

---

## 4. 角色与权限

| 角色 | 能力 |
|------|------|
| **群主 / 群管理员** | 「我的 → 活动运营」；创建/编辑自己的活动；受众仅能选有 staff 权的群（**可多选**）；推荐位；模板与积木；查看本活动互动数据 |
| **平台超管** | 以上 + 全站受众 + Hero B + 平台模板上架 + 内容库系列 |
| **群成员（受众）** | 看见曝光；阅读日课；点赞/评论等；分享（若开启） |
| **非成员** | 首页不出现；打开分享链见范围说明 |

- 一期编辑权：**创建者 + 超管**  
- 保存校验：每个所选群，创建者须仍为 owner/admin  
- **本期不做**群内活动入口

## 5. 入口与信息架构

```text
我的
 └─ 活动运营
      ├─ 活动列表
      ├─ 新建（选模板向导）
      └─ 我的模板

平台 /admin · 运营
 ├─ 全站活动 / Hero 主推（现有 Hero B 并入）
 ├─ 平台模板库
 └─ 内容库 · 系列（可选供给）
```

活动编辑 Tab：`基础` | `受众` | `曝光` | `落地页` | `数据`（可后置）

## 6. 受众

| 档位 | 可见范围 | 谁可配 |
|------|----------|--------|
| **指定群（默认）** | 所选群成员并集 | 群主/管理员 |
| **仅自己预览** | 创建者（+ 超管预览链） | 创建时可用 |
| **全站** | 全部用户 | **仅超管** |

- 未选群不可发布  
- UI 文案：「谁能看见」；群主侧不默认展示「全部用户」  

---

## 7. 曝光

| 挂载 | 配置 | 开放 |
|------|------|------|
| **今日推荐 · 第 1/2/3** | 可视化选位 | 群主 + 超管 |
| **Hero B** | 图、badge、暗色图 | **仅管理后台 · 与活动运营分离**；活动不挂载 Hero |

规则：

- 新建默认挂 **今日推荐 · 第 1**  
- 时段/受众在活动级统一（一期不按挂载拆）  
- 首页组装时 **按当前用户所属群过滤**  
- 推荐位次在「该用户可见的运营卡」内排序，避免他群活动占坑却不可见  
- 同位置冲突：按 priority；保存时提示

## 8. 模板是什么 · 活动人员怎么配置

### 8.0 模板是什么

**模板 = 预先搭好的一页活动骨架**（含若干积木控件、必填槽位、建议互动），不是上架内容包，也不是写死的功能页。

| | 模板做什么 | 模板不做什么 |
|--|------------|--------------|
| 结构 | 放好封面、标题、日课/代祷/RSVP 等控件位置 | 不替你写 10 天正文 |
| 引导 | 标出必填槽（如「活动标题」「封面图」） | 不含具体群、不含真实日程数据 |
| 起点 | 选中后生成可改的草稿页 | 不强制绑平台系列 |

活动人员：**选模板 → 改成自己的文案/图/日课 → 发布**。  
熟手可再拖拽加减控件，或把改好的页 **另存为我的模板** 下次复用。

### 8.0.1 活动人员配置流程（逐步）

```text
我的 → 活动运营 → 新建
  ① 选场景模板（按下表类型选，或选「空白」）
  ② 谁能看：多选自己有管理权的群（必填）
  ③ 何时：开始 / 结束
  ④ 改页面：填槽位文案、上传图、按天贴资料、开点赞/评论等
  ⑤ 曝光：默认「今日推荐 · 第 1」（主卡），可改第 2/3；不挂载 Hero B
  ⑥ 预览（仅自己）→ 对群成员发布
```

回改时用 Tab：`基础` | `受众` | `曝光` | `落地页`。  
**群内没有活动入口**；配置只在「活动运营」。

### 8.1 教会活动类型图谱

下列类型可用「模板 + 控件」配置，无需开发改版。（已去掉意见治理类、宣教外展类专题。）

| 类型域 | 典型活动 | 用户目标 | 用哪个模板 | 活动人员主要填什么 |
|--------|----------|----------|------------|-------------------|
| **A. 读经与灵修** | N 日教材、一书一章、节期灵修、背诵挑战 | 按天读、有进度 | N 日阅读 / 经文日 / 背诵挑战 | 按天粘贴资料与经文；或一日经文+短文 |
| **B. 代祷与关怀** | 代祷周、危机代祷、感恩记念 | 提交意向、彼此代祷 | 代祷召集 | 说明文案；开代祷意向；是否仅管理可见 |
| **C. 聚会与节奏** | 主日/祷告会/小组/线上聚会 | 知时知地、确认出席 | 聚会通知 | 时间地点或会议说明；开 RSVP；可选分享 |
| **D. 招募与服事** | 诗班/接待/儿童事工等岗位 | 报名岗位 | 服事招募 | 岗位名称与名额；说明；可选提问箱 |
| **E. 迎新与培育** | 欢迎新人、慕道/洗礼预习 | 认识群体、完成预习 | 迎新 / N 日预习 | 介绍文案；群入口；或按天预习材料 |
| **F. 节期与庆典** | 圣诞/复活/周年、感恩礼拜预告 | 氛围+出席+分享 | 节期问候 / 聚会通知 | 节期图文；需要出席则加日程+RSVP |
| **G. 见证与回应** | 见证征集、主日回应 | 写下/阅读见证 | 见证墙 | 征集说明；开见证投稿与点赞 |
| **H. 挑战与习惯** | 七日清晨读经、禁食说明、家庭祭坛 | 每日坚持 | N 日阅读 / 轻动员 | 日课或动员文案 + 打卡 |

**不纳入活动模板：** 在线奉献/收款、付费票务、复杂核销、人事纪律、意见投票/治理表决、宣教外展专题投放、外部完整 LMS。

### 8.2 平台场景模板清单

| 优先级 | 模板 | 名称 | 预置了什么（活动人员打开后看到） | 适合 |
|--------|------|------|----------------------------------|------|
| **MVP** | `promo` | 轻动员 | 封面、标题、正文、主按钮、点赞 | 一句话号召、功能引导 |
| **MVP** | `verse_day` | 经文日 | 经文引用、短文、打开圣经、点赞、评论 | 单日经文分享 |
| **MVP** | `multi_day` | N 日阅读 | 导语、日课列表（可选 7/10/14 天空槽）、进度、评论 | **10 天资料等按天阅读** |
| **MVP** | `prayer_drive` | 代祷召集 | 说明、代祷意向、阿们/点赞 | 代祷周 |
| **MVP** | `gathering` | 聚会通知 | 日程信息、说明、RSVP、分享 | 小组/主日/线上聚会 |
| **MVP** | `hub` | 多入口 | 标题、2～4 个入口卡 | 一页分发多个去处 |
| V1.1 | `memory` | 背诵挑战 | 背诵清单、打卡、分享 | 金句背诵 |
| V1.1 | `welcome` | 迎新 | 封面、介绍、群入口、提问箱 | 新人欢迎 |
| V1.1 | `testify` | 见证墙 | 说明、见证投稿、点赞 | 征集/展示见证 |
| V1.2 | `serve` | 服事招募 | 说明、名额报名、提问箱 | 岗位报名 |
| V1.2 | `season` | 节期问候 | 大图、短文、分享、可选 RSVP | 节期氛围 |
| 可选 | `blank` | 空白 | 仅标题 | 熟手从零拖控件 |

模板选择页按 **类型域 A–H** 分组。开发交付模板；活动人员只选模板、填内容、发布。

### 8.3 模板能力（产品增强项）

- 必填槽位引导（标题/封面/主按钮等）  
- 可选「结构锁定」（只改文案）  
- 可带建议推荐位；**不含具体群**  
- 平台库由超管上下架；群主可 **另存为我的模板**  
- 「连接系列」不作为默认推荐模板  

新建主路径：

```text
选模板 → 受众/时段 → 改槽位与积木（含日课） → 推荐位 → 预览 → 发布
```

## 9. 控件库

### 9.1 分类

| 类别 | 控件 |
|------|------|
| **内容** | 封面图、标题/副标题、正文、图片、分割线、版权 |
| **圣经** | 经文引用、打开圣经、**背诵清单**（多节可勾选记住） |
| **行动** | 主/次按钮、入口卡片组 |
| **可选连接** | 系列入口、计划入口、群入口（「去群里讨论」） |
| **多日** | **日课列表 / 按天阅读**（§10） |
| **聚会** | **日程信息**（时间/时区/地点或线上说明）、**倒计时**（可选） |
| **报名** | **RSVP**（出席/请假/未定）、**名额报名**（岗位+名额上限） |
| **互动** | 点赞、表情反应、评论、留言墙、打卡、代祷意向、提问箱、**见证投稿** |
| **传播** | **分享**（§11） |
| **数据** | 参与人数、RSVP 统计、报名剩余名额（只读） |

约束：

- 纵向积木排序，非自由画布；禁止任意 HTML / 外链脚本  
- 深链走现有白名单；会议链/地图链可用「按钮 · 外链」且需明示  
- 单页重互动控件建议 ≤ 2（日课内按天评论与页级点赞可并存）  
- UGC 对齐现有举报/审核思路  
- **不做投票/表决类控件**（意见与治理不在本产品范围）  

### 9.2 互动与报名控件摘要

| 控件 | 用户行为 | 配置要点 |
|------|----------|----------|
| 点赞 | 赞/取消 | 文案、是否显示总数 |
| 表情反应 | 选一表情 | 预设表情组（含「阿们」场景） |
| 评论 | 短评列表 | 字数、是否审核、仅受众可写 |
| 留言墙 | 展示型评论 | 条数、排序 |
| 打卡 | 「今日已读/我参加了」 | 每日一次或活动一次 |
| 代祷意向 | 提交短文 | 公开或仅管理可见 |
| 提问箱 | 向 staff 提问 | 仅 staff 可见与回复 |
| **见证投稿** | 提交短见证（可审核后展示） | 是否审核、字数、是否匿名 |
| **RSVP** | 出席 / 请假 / 未定 | 是否显示统计给成员 |
| **名额报名** | 选岗位报名/取消 | 岗位列表、每岗名额、截止 |
| **日程信息** | 只读展示 | 开始结束时间、地点/线上文案 |
| **背诵清单** | 勾选「已记住」 | 经文列表（书卷章节或粘贴） |

写权限与活动受众一致（所选群成员）。RSVP/报名数据一期可仅创建者见明细。

## 10. 多日日课（核心补齐）

支撑「10 天资料 → 首页发布 → 引导阅读」的主能力。

### 10.1 配置（运营）

- 模板：**N 日阅读**（可选 7 / 10 / 14 / 自定义天数）  
- 每天可配：`day`、标题、正文、图片、经文引用、打开圣经、**当日讨论引导句**、可选行动按钮  
- 解锁策略（活动级）：  
  - **按开始日解锁**（第 1 天 = 活动开始日，之后每日 +1）  
  - 或 **全部可读**  
- 进度：展示「已读 x / N」；支持「标记今日已读」打卡  

### 10.2 消费（用户）

- 落地页：封面/导语 + **日课列表** + 进度  
- 进入单日：阅读材料 → 打开圣经 → （可选）当日评论/讨论引导  
- 未解锁日：可见标题，内容锁定（若启用按日解锁）  

### 10.3 与系列内容库关系

| | 多日日课 | 平台系列包 |
|--|----------|------------|
| 谁生产 | 群主在活动内粘贴/上传 | 超管内容库上架 |
| 结构 | 轻量日课字段 | 完整 letter/workbook 等 |
| 用途 | **默认多日运营路径** | 可选「系列入口」跳转 |

**10 天资料标准打法**

1. 选模板「10 日阅读」  
2. 多选群；定开始/结束  
3. 按天填入资料与经文、讨论引导  
4. 加页级点赞 + 按天或页级评论；开启分享  
5. 挂今日推荐第 1 → 预览 → 发布

## 11. 分享（核心补齐）

| 项 | 规格 |
|----|------|
| 入口 | 页级分享按钮和/或「分享」控件；日课详情可带 `day` |
| 产物 | 链接 + 分享卡片（标题/图/摘要，可配） |
| 成员打开 | 进入活动落地页（可定位到某日） |
| 非成员打开 | 「仅对指定群开放」说明 + 登录/加群引导；**不泄漏正文** |
| 未登录 | 先登录再按受众鉴权 |

分享与复制链接均走同一鉴权。

| 群内活动 Feed | **本期不做** |

## 13. 状态机

```text
草稿 → 预览（仅自己）→ 已发布 → 已结束
                ↘ 停用（手动下线）
```

- 到点自动上/下线曝光  
- 已结束：曝光消失；落地展示结束态（可只读回顾，策略固定）  
- 支持复制活动、延期

## 14. 与现有能力关系

| 现有 | 关系 |
|------|------|
| Hero B | **独立**：管理后台配置全站主视觉；**活动运营不挂载**；活动走今日推荐三卡 |

| `/devotionals` 系列 | 可选连接；非主路径 |
| 深链白名单 | 按钮/入口复用 |

## 15. 客户端要点

1. 首页组装带用户群籍，过滤活动挂载  
2. 点击 → `/campaigns/:id`（可 `?day=`）按积木 + 日课渲染  
3. Web / App 行为一致  
4. 预览链可无视时段看草稿（创建者/超管）

## 20. 成熟产品优化建议（定稿补充）

目标：从「功能清单正确」进到「可规模化运营、可信任、可度量」。下列建议按优先级纳入迭代，不改变 §1～§19 的产品边界。

### 20.1 产品收束：一条主航道 + 清晰副线

| 建议 | 做法 |
|------|------|
| **主航道写死** | 对外叙事固定为：**群定向 · 模板自配 · 首页触达 · 阅读/回应**。避免同时讲「运营中台 / 教会 ERP / 建站工具」 |
| **MVP 再砍尖** | 首发只保证 3 条闭环跑通：① N 日阅读 ② 聚会通知+RSVP ③ 代祷召集。复杂自由积木可后置；`hub` 已在 V1.3 以轻量入口组落地 |
| **模板少于控件** | 先把 6 个 MVP 模板打磨「填完就能发」，再扩控件；忌控件很多、模板很糙 |
| **系列保持可选** | 配置页不出现「请先上架系列」；进阶才显示「连接系列」 |

### 20.2 活动人员体验（配置侧成熟度）

| 建议 | 做法 |
|------|------|
| **双模式** | 新手：三步向导（模板→群→发布）；熟手：Tab 全量编辑。默认进新手 |
| **发布检查清单** | 发布前一页清单：受众、时段、推荐位、封面、主行动、（日课）至少 1 天有内容；缺项不可发并定位到字段 |
| **真机预览优先于抽象预览** | 预览 = 成员看到的首页卡 + 落地页，含「非成员将看不到」提示 |
| **日课编辑体验** | 支持：按天复制、从上一活动复制、批量粘贴（纯文本按日分割）、单日预览；忌只能逐字段点选 10 次 |
| **草稿自动保存** | 本地/云端草稿，防长文丢失 |
| **复制 / 延期 / 再发布** | 成熟运营高频动作，V1.1 必做，避免每次从空白重建 |
| **空态教育** | 无群时引导「先成为某群管理员」；列表空态给 3 个示例活动（只读演示） |
| **权限文案** | 全程说「谁能看见」，不说 audience/staff |

### 20.3 成员体验（消费侧成熟度）

| 建议 | 做法 |
|------|------|
| **首页卡信息密度** | 推荐卡固定：标题 + 一句副文 + 类型标签（阅读/聚会/代祷）+ 可选进度「3/10」；忌运营随意堆文案导致卡高失控 |
| **落地页第一屏** | 只保留：标题、今日该做什么（今日日课 / RSVP / 代祷）、主按钮；长说明折叠 |
| **多日阅读节奏** | 默认「今日」大按钮；列表为辅；已读/未读状态清晰；按日解锁时给出「明天 X 点更新」 |
| **讨论克制** | 默认「按天评论」或页级二选一，避免双评论区；敏感代祷默认「仅管理可见」 |
| **结束态友好** | 已结束仍可只读回顾日课；RSVP/报名关闭并说明 |
| **通知（可选后置）** | 活动开始日、日课解锁、聚会前 2 小时：走现有推送体系且尊圣经 Tab 勿扰 |

### 20.4 模板体系成熟化

| 建议 | 做法 |
|------|------|
| **模板质量门禁** | 每个平台模板附：适用说明、填写示例、预览截图、预计填写时长 |
| **槽位语义化** | 用「今日经文说明」而非「正文 2」；降低教会用户理解成本 |
| **结构锁定可选** | 多群共用同一版式时，锁定布局只改文案，保证品牌一致 |
| **我的模板** | 另存时允许改名/分组；限制数量防垃圾模板 |
| **模板与数据分离** | 换模板前提示「内容可能需重填」；尽量做字段映射减少丢失 |

### 20.5 数据闭环（没有数据不算成熟运营产品）

MVP 即可做轻量，不必上复杂分析：

| 角色 | 看见什么 |
|------|----------|
| 活动人员 | 曝光人数（去重）、打开人数、日课已读人数、RSVP 统计、评论/代祷条数 |
| 超管 | 以上 + 模板使用占比、发布成功率、违规举报 |

列表页展示「打开率 / 已读」一列，驱动「发了有没有人看」。

### 20.6 信任、安全与牧养边界

| 建议 | 做法 |
|------|------|
| **UGC 默认安全** | 见证/代祷/评论：可「先审后发」开关；默认代祷仅管理可见 |
| **举报复用** | 评论/见证走现有举报与审核台，不另起一套 |
| **分享鉴权** | 非成员永不渲染正文；防链接转发泄密（尤其代祷） |
| **外链克制** | 会议链/地图链二次确认；记录创建者 |
| **内容版权** | 日课粘贴区保留「请确认使用授权」弱提示（对标创世记系列版权意识） |

### 20.7 与首页 / 群的关系（避免产品互相打架）

| 建议 | 做法 |
|------|------|
| **推荐位预算** | 单用户同时可见运营活动 ≤ 2（第 3 位谨慎）；多活动抢位规则产品化文案说明 |
| **不替代群任务** | 聚会 RSVP ≠ 群任务完成；文案区分「活动报名」与「群打卡」 |
| **去群讨论是跳转不是双写** | 点击进 IM，不在活动与群之间同步两套评论 |
| **Hero 纪律** | 群主永不可抢全站 Hero；超管 Hero 与群活动在用户侧互不污染 |

### 20.8 工程与产品架构（支撑成熟度）

| 建议 | 做法 |
|------|------|
| **活动为唯一聚合根** | 曝光、落地、互动、日课均挂 `campaign_id`，避免 Hero/推荐/页各一套配置 |
| **积木 Schema 版本化** | 控件 JSON 带 `schema_version`，旧活动可渲染可迁移 |
| **服务端鉴权** | 受众校验必须在 API，不只藏 UI |
| **双端同构** | Web/App 同一套 block 渲染协议，禁止一端特例模板 |
| **观测** | 发布失败、预览失败、日课加载失败进日志/告警 |

### 20.9 建议的「成熟 MVP」范围（相对 §16 的收紧）

若要以成熟感首发，建议首发范围：

1. 入口 + 多群受众 + 今日推荐（默认第 1）  
2. 模板仅 3 个打磨透：`multi_day` / `gathering` / `prayer_drive`（`promo`/`verse_day` 作轻量附赠）  
3. 日课 + 点赞/评论 + RSVP/日程  
4. 发布检查清单 + 预览 + 复制活动  
5. 轻数据：打开 / 已读 / RSVP  

背诵、见证、名额报名、`hub`、外链确认、到期自动结束、首页推荐接线（最多 3 卡）→ 已按 §16 V1.1–V1.4 落地（Web）。活动运营不挂载 Hero；Hero B 仍由管理后台独立配置。

### 20.10 成功指标（北星）

| 指标 | 含义 |
|------|------|
| 周活跃配置者 | 有发布/更新活动的群主·管理员数 |
| 活动打开率 | 打开 / 曝光（群成员） |
| 日课完成率 | 活动结束时平均已读天数 / 总天数 |
| RSVP 响应率 | 有表态人数 / 曝光人数 |
| 配置时长 | 从新建到首次发布中位数（目标 &lt; 15 分钟，N 日阅读可放宽） |
| 支持工单 | 「不会配 / 配坏了」类反馈下降 |

### 20.11 优化原则（验收用）

1. **少模板、高质量**，优于多模板半成品  
2. **发布有检查、消费有今日焦点、运营有数据**  
3. **安全默认偏紧**（群定向、代祷私密、分享不泄文）  
4. **主航道稳定**，新类型用新模板扩展，不改信息架构  
5. **每个版本只承诺可演示的完整闭环**
